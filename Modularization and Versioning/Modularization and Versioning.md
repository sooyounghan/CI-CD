-----
### CI / CD 모듈화 및 버저닝
-----
1. 모듈화 및 버전 관리
2. 모듈화 : 작성한 코드를 재사용 가능한 독립적인 단위로 분리
   - 예) module : image-build, deploy
3. 버저닝 : 작성한 코드의 다양한 버전을 관리하는 것으로, 프로젝트의 안정성을 유지하고, 이전 버전으로 쉽게 Rollback할 수 있음
   - 예) image-build : v1.0.0, v2.0.0 / deploy : v1.0.0, v2.0.0

4. Github Actions 모듈화, 버저닝
   - 필요성 및 구성 방법
   - CI / CD 시나리오 4에 반영
