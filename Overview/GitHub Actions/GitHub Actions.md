-----
### Github Actions
-----
1. GitHub의 자동화 도구
   - 단순한 워크플로우에서부터 복잡한 워크플로우까지 구축 가능
   - 소프트웨어 개발 워크플로우 자동화 도구

2. 특징
   - 개발 / 테스트 / 배포 등의 워크플로우를 GitHub 저장소 내에서 직접 관리
   - GitHub 저장소와 완전히 통합되어, 추가적인 인프라 설정 및 외부 도구로 관리할 필요가 없음
   - YAML을 사용하여, 워크플로우를 쉽게 설정 가능
     + YAML : 데이터 구조를 간단하게 표현한 직렬화 언어 (.yml, .yaml)

3. 다양한 플랫폼 지원 (Linux, Windows, MacOS에서 모두 실행 가능)
4. 다양한 이벤트에 따라 실행하는 워크플로우 : GitHub Event를 기반으로 워크 플로우를 실행할 수 있음
   - GitHub 이벤트를 기반으로 워크플로우를 트리거 하는 방식은 큰 장점
   - 이런 특성 덕분에 Push나 Pull Request 외에도 GitHub에서 발생한 다양한 이벤트의 반응에서 워크플로우 실행 가능
   - 실제 이러한 기능을 활용해 복잡한 워크플로우를 구성하거나 특정 조건에 따른 다양한 태스크를 자동화하는데 활용하는 일이 많음
